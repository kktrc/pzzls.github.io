解析仿礼物说这个App

Jul 18, 2016

Read the Fucking Source Code!Linus在回答有人提出的Minix的一个问题时，第一句话就是RTFSC，这就是这句话的由来。Source Code直接反映了一个代码的运行方式，阅读源码永远是最有效的学习方式。今天我只是想作为一个观众解析一下这个源代码，也学习一下怎么样阅读源码。

github仓库地址

github

首先让我们来看一下目录结构

很标准的iOS工程目录:

  • Liwushuo.xcodeproj:我叫他工程文件，是我们创建iOS工程时创建的目录，主要内容是关于项目的设置，依赖等等。
  • Liwushuo.xcworkspace: 这个cocoapods为我们创建的工程目录，内容也差不多。
  • Liwushuo: 这里是源码目录，我们的源码一般都放在这里。
  • Pods:我们依赖的第三方工程的目录。
  • .gitignore: git的忽略文件
  • Podfile:cocoapods的依赖配置文件
  • Podfile.lock:cocoapods的依赖锁定文件
  • README.md:README

接下来我们看一下Podfile这个文件，看一下这个工程依赖了哪些第三方工程库:

  • SnapKit //A Swift Autolayout DSL for iOS & OS X
  • MJRefresh //An easy way to use pull-to-refresh.
  • SVProgressHUD //A clean and lightweight progress HUD for your iOS and tvOS app.

然后我进入目录执行了pod install(ps: 他这里的pod的版本比较低，应该是0.39，而现在已经是1.0.1了所以我切换了ruby环境用了ruby 2.2.5下的pod 0.39来执行)。
